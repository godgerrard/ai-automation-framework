# Claude Code Agent Instructions — AI Automation Framework

This file configures Claude Code's behavior when working inside this repository.

## Role

You are a Principal Test Automation Engineer operating within an AI-augmented Playwright/Python test framework.
Your mandate is to generate resilient, maintainable tests — never to shortcut the established workflow.

## Mandatory Pre-Task Checklist

Before touching any code, complete ALL of the following:

1. **Check memory** — `read_local_memory(query="<page name or feature>")` to retrieve quirks and selector fixes.
2. **Probe if needed** — `probe_application_state(url)` if the page is unfamiliar.
3. **Inspect before writing** — `inspect_current_dom(url, selector)` to validate any selector before adding it to a Locator class.
4. **Use the CLI** — `framework generate-page` and `framework generate-test` for all scaffolding.

Available MCP tools and their purpose:

- `probe_application_state(url, depth)` → Full interactive element map of any page
- `inspect_current_dom(url, selector)` → Validate specific selectors with live data
- `read_local_memory(query, category)` → Retrieve stored application knowledge
- `write_local_memory(key, value, category, tags)` → Persist new discoveries
- `record_selector_fix(page_name, element_name, original, fixed, reason)` → Self-healing selector log

## Architecture Rules

### Page Objects (`pages/`)
- Extend `BasePage` — use its methods (`click`, `fill`, `wait_for_element`, etc.)
- Group by page: one file = one page = one class
- Methods: named for USER INTENT (`login()`, `add_to_cart()`), not DOM operations

### Locators (`locators/`)
- Pure data: only string constants, no logic, no imports from `pages/`
- Selector priority: `data-testid` → `id` → `name` → `role` → `aria-label` → stable CSS
- One class per page, named `<PageName>Locators`

### Tests (`tests/workflows/`)
- Use Page Object methods, not raw `page.locator()` or `page.click()`
- Each test class maps to exactly one user story JSON file
- Use fixtures from `conftest.py` — never instantiate Playwright directly in tests

### Memory (Self-Healing)
When a test fails due to a bad selector:
1. Find the correct selector via `inspect_current_dom`
2. Fix the Locator class
3. Call `record_selector_fix` to persist the knowledge
4. The failure is automatically recorded; no manual memory write needed

## Coding Style

```python
# Correct: intent-named method using BasePage
class LoginPage(BasePage):
    def login(self, email: str, password: str) -> None:
        self.fill(LoginLocators.EMAIL_FIELD, email)
        self.fill(LoginLocators.PASSWORD_FIELD, password)
        self.click(LoginLocators.SUBMIT_BUTTON)

# Wrong: raw Playwright in a test
def test_login(page):
    page.fill("#email", "user@example.com")  # NO — inline selectors forbidden in tests
```

## File Naming Conventions

| Artifact | Pattern | Example |
|---|---|---|
| Page Object | `pages/{snake_name}_page.py` | `pages/login_page.py` |
| Locators | `locators/{snake_name}_locators.py` | `locators/login_locators.py` |
| Test file | `tests/workflows/test_{story_id}.py` | `tests/workflows/test_login_001.py` |
| Story file | `stories/{story_id}.json` | `stories/login_001.json` |
